# 📊 Веб-сервис для генерации отчетов / конвертации файлов

[![Django CI](https://github.com/aloecC/file_converter/actions/workflows/main.yml/badge.svg)](https://github.com/aloecC/file_converter/actions/workflows/main.yml)

 Описание проекта

Этот проект представляет собой высокопроизводительный веб-сервис для асинхронной обработки файлов и генерации пользовательских отчетов. Пользователи могут загружать файлы (CSV, Excel) или предоставлять параметры в формате JSON, а сервис в фоновом режиме выполнит обработку (например, конвертацию, фильтрацию данных, агрегацию) и предоставит результат для скачивания в формате Excel.
Ключевая особенность: Асинхронная обработка с использованием Celery и Redis, что позволяет сервису быстро отвечать на запросы и выполнять ресурсоемкие задачи в фоне, не блокируя основной API.

 🌟 Возможности

•  Загрузка файлов: Поддержка CSV и Excel (XLS/XLSX) файлов для обработки.
•  Генерация отчетов по параметрам: Возможность создавать отчеты, передавая данные и параметры фильтрации непосредственно в JSON-формате, без загрузки файла.
•  Асинхронная обработка: Все операции по обработке файлов выполняются в фоновом режиме с использованием Celery, обеспечивая отзывчивость API.
•  Отслеживание статуса: Пользователи могут отслеживать статус выполнения своего запроса на отчет (PENDING, PROCESSING, SUCCESS, FAILED).
•  Скачивание результатов: После успешной обработки предоставляется прямая ссылка для скачивания готового отчета в формате Excel.
•  Пользовательская аутентификация: Все запросы на генерацию отчетов привязаны к аутентифицированному пользователю.
•  Контейнеризация: Весь проект упакован в Docker-контейнеры для легкого развертывания и масштабирования.

 🛠️ Используемые технологии

 🚀 Архитектура проекта

Проект построен по принципу микросервисной архитектуры (в контексте Docker-контейнеров) с асинхронной обработкой задач:

1. Пользователь отправляет запрос (с файлом или параметрами) через REST API.
2. Django API (DRF) принимает запрос, аутентифицирует пользователя, сохраняет метаданные запроса и информацию о файле в PostgreSQL и Media Storage (общая папка для файлов).
3. Django API немедленно ставит задачу на обработку в Redis (брокер сообщений Celery) и возвращает пользователю статус 202 Accepted и PENDING.
4. Celery Worker (работающий в отдельном контейнере) извлекает задачу из Redis.
5. Celery Worker читает входной файл (или параметры) из Media Storage, использует Pandas для обработки данных.
6. После обработки, Celery Worker сохраняет сгенерированный отчет в Media Storage и обновляет статус запроса (SUCCESS или FAILED) и путь к выходному файлу в PostgreSQL.
7. Пользователь периодически запрашивает статус своего отчета. Когда статус становится SUCCESS, API предоставляет прямую ссылку для скачивания готового файла из Media Storage.

 ⚙️ Установка и настройка

1. Клонируйте репозиторий
2. Установите зависимости
3. Настройте переменные окружения:
  Создайте файл .env в корне проекта или пропишите в settings.py
    TELEGRAM_BOT_TOKEN=your_token_here
    SECRET_KEY=django_secret_key
    DEBUG=True
    CELERY_BROKER_URL=redis://localhost:6379/0
4. Примените миграции
5. Создайте суперпользователя (для доступа к админке)

🏃 Шаги для Запуск проекта через
docker-compose:

1. Копирование Dockerfile и Docker-compose.yml
2. Настройка файла .env (Конфиденциальность)
3. Проверка файлов исключений .gitignore и .dockerignore
Первый запуск
1. docker compose down -v # Остановит и удалит все контейнеры, сети, и ТОМЫ (postgres_data, media)
2. docker compose up -d --build # Запустит все сервисы и соберет образы
*  -d: Запустить в фоновом режиме.
  •  --build: Пересобрать образы (особенно важно после изменений в Dockerfile).
3. docker compose ps # Проверит статус контейнеров(Все сервисы должны быть в статусе Up или Running)
4. docker compose logs -f  # Покажет логи (для отладки)
5. docker compose exec app python manage.py migrate # Выполнит миграции Django
6. docker compose exec app python manage.py createsuperuser # Создаст суперпользователя Django
7. Проверьте веб-приложение:
  Откройте браузер и перейдите на http://localhost:8000/. Должна открыться страница Django.
  Проверьте админку: http://localhost:8000/admin/.
8. docker compose logs worker # Проверит Celery Worker



🚀 Использование API

Все API-эндпоинты требуют аутентификации. Используйте Basic Auth (логин/пароль суперпользователя) в Postman или аутентифицируйтесь через админ-панель Django в браузере.

Базовый URL API: http://127.0.0.1:8000/api/reports/

 1. Создание отчета по параметрам (без файла)

•  Эндпоинт: POST /api/reports/
•  Тип запроса: application/json
•  Пример тела запроса:
      {
        "parameters": {
            "start_date": "2023-01-01",
            "end_date": "2023-12-31",
            "filters": {
                "category": "Electronics",
                "min_price": 100
            }
        }
    }
*  Ожидаемый ответ (201 Created):
*     {
        "id": 1,
        "user": "admin",
        "uploaded_file": null,
        "parameters": {
            "start_date": "2023-01-01",
            "end_date": "2023-12-31",
            "filters": {
                "category": "Electronics",
                "min_price": 100
            }
        },
        "task_id": "a1b2c3d4-e5f6-7890-1234-567890abcdef",
        "status": "PENDING",
        "output_file": null,
        "created_at": "2024-03-07T10:00:00.123456Z",
        "updated_at": "2024-03-07T10:00:00.123456Z"
    }
*
### 2. Загрузка файла для обработки

•  Эндпоинт: POST /api/reports/upload_file/
•  Тип запроса: multipart/form-data (в Postman выберите form-data)
•  Поля формы:
  •  file: Выберите ваш файл (CSV или XLSX).
  •  parameters (опционально): JSON-строка с дополнительными параметрами для обработки.
•  Пример запроса в Postman:
  •  Key: file, Value: (выберите файл your_data.csv)
  •  Key: parameters, Value: {"filter_column": "Product", "filter_value": "Laptop"}, Type: text
•  Ожидаемый ответ (202 Accepted):
      {
        "id": 2,
        "user": "admin",
        "uploaded_file": 123, # ID объекта UploadedFile
        "parameters": {
            "filter_column": "Product",
            "filter_value": "Laptop"
        },
        "task_id": "fedcba98-7654-3210-fedc-ba9876543210",
        "status": "PENDING",
        "output_file": null,
        "created_at": "2024-03-07T10:05:00.123456Z",
        "updated_at": "2024-03-07T10:05:00.123456Z"
    }

### 3. Получение статуса отчета и скачивание результата

•  Эндпоинт: GET /api/reports/{id}/status/
  (Замените {id} на ID отчета из ответа POST-запроса)
•  Пример URL: GET http://127.0.0.1:8000/api/reports/2/status/
•  Ожидаемый ответ (200 OK):
  •  Если в процессе:

        {
            "id": 2,
            "status": "PROCESSING",
            "task_id": "fedcba98-7654-3210-fedc-ba9876543210",
            "error_message": null,
            "output_file": null
        }

*  Если успешно завершен:

          {
            "id": 2,
            "status": "SUCCESS",
            "task_id": "fedcba98-7654-3210-fedc-ba9876543210",
            "error_message": null,
            "output_file": "http://127.0.0.1:8000/media/reports/report_2_20240307_100500.xlsx"
        }
*  Если произошла ошибка:

          {
            "id": 2,
            "status": "FAILED",
            "task_id": "fedcba98-7654-3210-fedc-ba9876543210",
            "error_message": "Формат файла не поддерживается. Используйте CSV или Excel.",
            "output_file": null
        }

* Скачивание файла: Если status равен "SUCCESS", скопируйте URL из поля output_file и вставьте его в адресную строку вашего браузера. Файл будет автоматически скачан.



 4. Просмотр списка всех запросов на отчеты

•  Эндпоинт: GET /api/reports/
•  Ожидаемый ответ (200 OK): Список всех запросов на отчеты, созданных текущим пользователем.
